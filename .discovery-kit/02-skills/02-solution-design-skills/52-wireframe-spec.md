# Навык: Wireframe Specification (Unicode Wireframes Rounded)

## Цель

Помочь визуализировать UI-логику через текстовые wireframes. Превратить user stories или описание задачи в набор экранов, готовых для передачи разработчику или дизайнеру.

---

> [!IMPORTANT]
> ## Standalone-скилл
>
> Этот скилл **не требует** предыдущих артефактов (Brief, IA, USM).
> Пользователь приходит с конкретной задачей — помочь визуализировать.
>
> **Типичные кейсы:**
> - «Мне нужна форма авторизации, не понимаю как её сделать»
> - «Есть сложная логика, не могу поставить задачу разработчику»
> - «Вот три user stories — как их правильно показать?»

---

## Входные данные

**Что принимаем:**
- User stories с acceptance criteria
- Текстовое описание задачи/фичи
- Скриншоты существующего UI (для редизайна)
- Ссылки на аналоги («хочу как в Notion»)

**Первый вопрос — платформа:**

```
Для какой платформы делаем wireframe?

1. [M] Mobile (375px width)
2. [D] Desktop (1200px+ width)
3. [M+D] Оба варианта (сначала mobile)
```

---

## Процесс

### Шаг 1: Анализ scope

После получения входных данных определи:

**Один экран или несколько?**

Признаки нескольких экранов:
- Разные состояния (до/после действия)
- Переходы между страницами
- Модальные окна / sidebars
- Несколько user stories

**Если несколько экранов:**

Сначала покажи схему взаимодействия:

```
╭─────────────╮    ╭─────────────╮    ╭─────────────╮
│   Экран 1   │───▶│   Экран 2   │───▶│   Экран 3   │
│   (Логин)   │    │  (Дашборд)  │    │  (Профиль)  │
╰─────────────╯    ╰─────────────╯    ╰─────────────╯
       │                                     │
       ╰──────────── Logout ─────────────────╯
```

Спроси: «Какой экран визуализируем первым?»

### Шаг 2: Итеративная визуализация

Для каждого экрана:

1. **Нарисуй wireframe** (см. формат ниже)
2. **Добавь аннотации** — что делает каждый элемент
3. **Спроси подтверждение** — «Всё верно? Что изменить?»
4. **Итерируй** до подтверждения
5. **Переходи к следующему** экрану

## Формат Unicode Wireframe (Rounded)

### Базовые элементы

```
╭──────────────────────────────────────╮
│             HEADER                   │
├──────────────────────────────────────┤
│                                      │
│            CONTENT                   │
│                                      │
├──────────────────────────────────────┤
│             FOOTER                   │
╰──────────────────────────────────────╯
```

### Библиотека компонентов

**Контейнеры:**
```
╭───────╮  Карточка (card)
│       │
╰───────╯

╭┄┄┄┄┄┄┄╮  Секция с пунктиром (section)
┆       ┆
╰┄┄┄┄┄┄┄╯

╭━━━━━━━╮  Модальное окно (modal) — жирная линия
┃       ┃
╰━━━━━━━╯
```

**Формы:**
```
╭─────────────────╮
│ Placeholder...  │  Input field
╰─────────────────╯

╭─────────────────╮
│ Option 1      ▼ │  Dropdown/Select
╰─────────────────╯

(○) Unselected        Radio
(●) Selected          Radio (selected)
[ ] Unchecked         Checkbox
[✓] Checked           Checkbox (checked)

╭─────────────────╮
│                 │
│   Textarea      │  Multi-line input
│                 │
╰─────────────────╯
```

**Кнопки:**
```
╭─────────────╮
│   Primary   │  Primary button
╰─────────────╯

╭┄┄┄┄┄┄┄┄┄┄┄┄┄╮
┆  Secondary  ┆  Secondary/outlined
╰┄┄┄┄┄┄┄┄┄┄┄┄┄╯

[ Cancel ]        Text button/link
```

**Навигация:**
```
╭──────┬──────┬──────┬──────╮
│ Tab1 │ Tab2 │ Tab3 │ Tab4 │  Tabs
╰──────┴──────┴──────┴──────╯

☰ Menu                        Hamburger menu
← Back                        Back button
◎ _______________             Search bar
```

**Контент:**
```
╭─────╮
│ IMG │  Изображение
╰─────╯

╭─────┬─────────────────────╮
│ ◯   │ Username            │  Avatar + text
╰─────┴─────────────────────╯

████████████████████         Progress bar (full)
████████░░░░░░░░░░░░         Progress bar (partial)

────────────────────────     Divider/separator
```

**Иконки (Unicode — НЕ emoji):**
```
← → ↑ ↓ ↗ ↖    Стрелки
✓ ✗ ⚠ ℹ        Статусы
★ ☆            Рейтинг
◎ ◉ ○ ●        Круги (search, radio, bullets)
◇ ◆            Ромбы (notifications, markers)
⌂              Дом (home)
⚙              Настройки (settings)
◯              Аватар/профиль (user)
※              Пароль/защита (lock)
⋮ ⋯            Меню (vertical/horizontal dots)
＋ －           Добавить/удалить
☰              Hamburger menu
▼ ▲            Dropdown arrows
```

**Списки:**
```
├── Item 1
├── Item 2
╰── Item 3

• Bullet point
1. Numbered item
```

### Размеры для платформ

**Mobile (375px):**
```
╭────────────────────────────────────╮
│              38 chars              │
╰────────────────────────────────────╯
```

**Desktop (1200px+):**
```
╭────────────────────────────────────────────────────────────────────────────╮
│                                 76 chars                                    │
╰────────────────────────────────────────────────────────────────────────────╯
```

## Аннотации

Используй стрелки и пояснения справа или снизу:

```
╭──────────────────────────────────────╮
│  ⌂ AppName                     ◯ ⚙ ◇ │ ← Header: logo + user actions
├──────────────────────────────────────┤
│                                      │
│  ╭─────────────────────────────────╮ │
│  │ ◎ Search products...            │ │ ← Search: filters catalog
│  ╰─────────────────────────────────╯ │
│                                      │
│  ╭─────────╮  ╭─────────╮            │
│  │  IMG    │  │  IMG    │            │ ← Product cards grid
│  │ $99.00  │  │ $149.00 │            │
│  │ ★★★★☆   │  │ ★★★★★   │            │
│  ╰─────────╯  ╰─────────╯            │
│                                      │
╰──────────────────────────────────────╯

Интерактивность:
• Клик на карточку → страница товара
• Search → фильтрация в реальном времени
• ◯ → профиль пользователя
```

## Целевая структура вывода

Для каждого экрана генерируй блок:

```markdown
## Экран: [Название]

**Контекст:** [Когда пользователь видит этот экран]
**Цель пользователя:** [Что хочет сделать]

### Wireframe

[Unicode wireframe здесь]

### Аннотации

| # | Элемент | Поведение |
|---|---------|-----------|
| 1 | [Элемент] | [Что происходит при взаимодействии] |
| 2 | [Элемент] | [Что происходит при взаимодействии] |

### Состояния (если есть)

- **Empty state:** [Что показываем, если данных нет]
- **Loading:** [Индикатор загрузки]
- **Error:** [Сообщение об ошибке]

### Переходы

- [Действие] → [Куда ведёт]
```

## Пример: Форма логина (Mobile)

**Input от пользователя:**
> Мне нужна форма авторизации. Email + пароль. Есть ссылка на регистрацию и восстановление пароля.

**Output:**

## Экран: Login

**Контекст:** Пользователь не авторизован, открывает приложение
**Цель пользователя:** Войти в аккаунт

### Wireframe

```
╭────────────────────────────────────╮
│                                    │
│           ╭────────╮               │
│           │  LOGO  │               │
│           ╰────────╯               │
│                                    │
│      Welcome back!                 │
│      Sign in to continue           │
│                                    │
│  ╭────────────────────────────╮    │
│  │ @ Email address            │    │ ← 1
│  ╰────────────────────────────╯    │
│                                    │
│  ╭────────────────────────────╮    │
│  │ ※ Password            ◉    │    │ ← 2
│  ╰────────────────────────────╯    │
│                                    │
│           [ Forgot password? ]     │ ← 3
│                                    │
│  ╭────────────────────────────╮    │
│  │         Sign In            │    │ ← 4
│  ╰────────────────────────────╯    │
│                                    │
│  ─────────── or ───────────        │
│                                    │
│  ╭┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄╮    │
│  ┆  G  Continue with Google   ┆    │ ← 5
│  ╰┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄╯    │
│                                    │
│     Don't have an account?         │
│        [ Sign Up ]                 │ ← 6
│                                    │
╰────────────────────────────────────╯
```

### Аннотации

| # | Элемент | Поведение |
|---|---------|-----------|
| 1 | Email input | Валидация формата email. Иконка @ |
| 2 | Password input | ◉ toggle показывает/скрывает пароль. Иконка ※ |
| 3 | Forgot password | → экран восстановления пароля |
| 4 | Sign In button | Disabled пока поля пустые. Отправляет форму |
| 5 | Google OAuth | Открывает OAuth flow в popup/redirect |
| 6 | Sign Up link | → экран регистрации |

### Состояния

- **Error (wrong credentials):**
  ```
  ⚠ Invalid email or password
  ```
  Показывается над полем email, поля подсвечиваются красным

- **Loading:**
  Кнопка "Sign In" показывает spinner, disabled

### Переходы

- Sign In (success) → Dashboard
- Forgot password → Password Recovery
- Sign Up → Registration
- Google OAuth (success) → Dashboard

## Заметки

- **Один экран за раз** — не перегружай пользователя
- **Спрашивай подтверждение** после каждого экрана
- **Итерируй** — wireframe редко получается с первого раза
- **Без цветов** — только структура и поведение
- **Только Unicode** — никаких emoji (ломают ширину колонок)
- **Rounded corners** (╭╮╰╯) — более мягкий, современный вид
- Если пользователь не знает что хочет — предложи 2-3 варианта
