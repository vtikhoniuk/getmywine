# Brief: NutriBot

> **Версия:** 1.0
> **Дата:** 2026-01-30
> **Статус:** Утверждён для Discovery

---

## Что нужно сделать?

**Описание задачи:**
Создать AI-ассистента (чат-бота) для нутрициологов, который автоматизирует сбор информации о клиентах, анализирует их данные и подготавливает структурированные отчёты для специалиста.

**Job Story:**
Когда ко мне обращается новый клиент, я хочу, чтобы бот собрал всю необходимую информацию о его питании, здоровье и целях, проанализировал результаты анализов и подготовил структурированный отчёт, чтобы я мог сосредоточиться на принятии решений вместо рутинной работы по сбору данных.

---

## Полезное действие (Зачем?)

### Базовая проблема пользователя:

**Для нутрициолога:**
- Тратит 40-60 минут на первичный сбор анамнеза с каждым клиентом
- Вручную обрабатывает результаты анализов и сопоставляет с нормами
- Не может масштабировать практику — количество клиентов ограничено временем
- Информация о клиентах разбросана по разным источникам (мессенджеры, заметки, файлы)

**Для клиента нутрициолога:**
- Долгое ожидание ответа от специалиста
- Неудобство заполнения длинных анкет
- Отсутствие понимания, какие анализы сдавать
- Нет возможности получить быструю обратную связь

### Ценность для бизнеса:
- Увеличение пропускной способности нутрициолога в 2-3 раза
- Стандартизация процесса сбора данных
- Повышение качества данных для принятия решений
- Создание базы знаний по клиентам

## Жёсткие ограничения (Constraints)

### Бюджет:
- **CAPEX:** Не более 300 000 ₽ на MVP
- **OPEX:** Не более 30 000 ₽/месяц (включая LLM API)

### Сроки:
- **MVP:** 8 недель
- **Пилот с первыми клиентами:** 10 недель

### Команда:
- 1 full-stack разработчик
- 1 нутрициолог (доменный эксперт, part-time)
- Claude/GPT API для LLM-функциональности

### Инфраструктура:
- Интеграция с Telegram (основной канал коммуникации)
- Облачный хостинг (Yandex Cloud предпочтительно)
- Использование LLM API (Claude/GPT) для анализа и генерации

### Регуляторные:
- 152-ФЗ (персональные данные) — данные хранятся в РФ
- Врачебная тайна — бот не ставит диагнозы, только помогает собирать информацию
- Disclaimer: рекомендации носят информационный характер

## Что нужно учесть?

### Формат результата:
- Telegram-бот для взаимодействия с клиентами
- Веб-панель для нутрициолога (просмотр отчётов, управление клиентами)
- Экспорт отчётов в PDF

### Согласования:
- Утверждение скриптов диалогов с нутрициологом
- Валидация рекомендаций по анализам с медицинским консультантом
- Согласование текстов disclaimer'ов с юристом

---

## Необходимая информация

### От нутрициолога (доменного эксперта):
- Структура первичного опросника (какие вопросы задавать)
- Список базовых анализов по категориям целей
- Референсные значения анализов (или источники норм)
- Примеры готовых отчётов (как выглядит результат)
- Типичные паттерны и красные флаги в анализах

### Технические:
- Лимиты Telegram Bot API
- Ценообразование LLM API
- Требования 152-ФЗ для хранения медицинских данных

## Критерии приёмки

### MVP:
- [ ] Бот задаёт структурированные вопросы о питании, здоровье, целях
- [ ] Бот понимает свободный текст и извлекает структурированные данные
- [ ] Бот рекомендует список анализов на основе целей клиента
- [ ] Клиент может загрузить фото/PDF результатов анализов
- [ ] Система распознаёт и структурирует результаты анализов
- [ ] Нутрициолог видит карточку клиента с собранными данными
- [ ] Система генерирует черновик отчёта с выводами и рекомендациями
- [ ] Нутрициолог может редактировать и экспортировать отчёт

### Quality Gates:
- [ ] Точность извлечения данных из анализов > 95%
- [ ] Время сбора первичной информации < 15 минут
- [ ] Удовлетворённость нутрициолога качеством отчёта > 4/5

## Out of Scope

### Не входит в MVP:
- Видео-консультации
- Интеграция с лабораториями (автоматическое получение результатов)
- Мобильное приложение (только Telegram + веб)
- Составление готовых планов питания (только рекомендации)
- Трекинг соблюдения диеты
- Интеграция с фитнес-трекерами
- Групповые программы
- Marketplace нутрициологов
- Автоматическая запись на приём

### Сознательные ограничения:
- Бот не ставит диагнозы
- Бот не заменяет врача
- Бот не даёт медицинских назначений
- Финальное решение всегда за нутрициологом

## Открытые вопросы

1. **Источник норм анализов:** Откуда брать референсные значения? Использовать стандартные лабораторные нормы или собственную базу нутрициолога?

2. **Формат загрузки анализов:** Поддерживать только фото/PDF или также ручной ввод? Какие лаборатории наиболее популярны (для парсинга их форматов)?

3. **Персонализация:** Насколько бот должен адаптировать диалог под ответы клиента? Нужны ли ветвления по сценариям?

4. **Мультиязычность:** Нужна ли поддержка языков кроме русского?

5. **Уведомления:** Как напоминать клиенту о необходимости сдать анализы или заполнить информацию?

## Оценка рисков

### Продуктовые риски:
| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Низкая точность OCR анализов | Средняя | Высокое | Fallback на ручной ввод + итеративное улучшение |
| Клиенты не доверяют боту | Низкая | Высокое | Прозрачный disclaimer, бренд нутрициолога |
| LLM галлюцинации в выводах | Средняя | Высокое | Чёткие промпты, проверка нутрициологом |

### Бизнес-риски:
| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Нутрициологи не готовы платить | Средняя | Критическое | Пилот с 3-5 нутрициологами, валидация цены |
| Регуляторные ограничения | Низкая | Высокое | Консультация с юристом, conservative подход |

### Общий вывод:
Риски управляемые. Ключевой митигационный фактор — тесная работа с нутрициологом-экспертом на всех этапах и поэтапная валидация с реальными клиентами.
